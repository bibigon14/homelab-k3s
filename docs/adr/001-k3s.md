# ADR-001: k3s over full Kubernetes

Date: 2026-06
Status: Accepted

## Context

Needed container orchestration for Raspberry Pi 5 (8 GB RAM, ARM64). Options: Docker Compose, kubeadm, k3s, MicroK8s.

## Decision

Use k3s.

## Reasons

- Resource footprint: ~500 MB RAM vs ~2+ GB for a full control plane.
- ARM64 support: first-class, ships as a single binary.
- Production parity: same Kubernetes API - every manifest, Helm chart, and kubectl command works identically to production.
- Built-in defaults: Traefik, CoreDNS, local-path StorageClass, metrics-server all included.
- Single binary + systemd: easy recovery after reboot, no etcd cluster to manage.

## Consequences

- Single node = no HA. Mitigated by fast boot (~3 min recovery) and persistent-journal monitoring.
- local-path PVs are node-local. Migration path when second node arrives: Longhorn or Rook-Ceph.
- Images must be imported via k3s ctr images import, not docker push.
